# Chat via WebView DOM bridge with native rendering

Reddit Chat has no public API — no REST endpoints, no OAuth scopes, nothing. Every other third-party Reddit client skips it. fspez renders Chat using a hidden WebView that maintains a live WebSocket connection to Reddit's chat servers. JavaScript injected into the WebView extracts message data from the DOM and pipes it to native Flutter widgets via a JS bridge.

**Considered Options**:
- **Themed WebView (B2)**: Inject CSS to match the app's theme, but keep Reddit's own chat UI. Rejected because it would look out of place and break the native feel.
- **Skip Chat entirely**: What every other third-party client does. Rejected because the user explicitly requires it.

**Consequences**: The chat DOM is generated by webpack with hashed class names — stable selectors (ARIA roles, Redux store hooks, WebSocket interception) must be discovered and maintained. Every Reddit frontend deploy carries a non-zero risk of breaking extraction, requiring a rapid update. The hidden WebView consumes memory and battery; its lifecycle must be managed carefully to avoid OS process monitoring flags.
